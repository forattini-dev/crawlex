# Detection consolidation under `fingerprint::Fingerprinter`

Three independent detection paths existed in `crawlex` before PRD #25: `discovery::tech_fingerprint` (1312 LOC of CMS / framework / web-server / language matchers), `runner::ChallengeDetector` (218 LOC of antibot vendor signatures + JS-stub / noscript heuristics, ported in slice #19 from the older `escalation::detect_antibot_vendor`), and the `antibot::detect_*` + `antibot::block_detector` family (≈1000 LOC of antibot-only logic). The three drifted independently, used three different `Vendor`-shaped enums (`error::AntibotVendor`, `antibot::ChallengeVendor`, the implicit shape inside `runner::ChallengeSignal`), and could not share evidence — when Cloudflare was identified, two of the three couldn't see the other's evidence.

PRD #25 collapsed all three into one `fingerprint::Fingerprinter` engine with 18 plug-in `Source` impls organized into Hot / Warm / Cold tiers. The consolidated `fingerprint::detection::Vendor` enum spans 70+ variants across CDN / WAF / Antibot / CMS / Ecommerce / Frontend / Backend / Analytics / Auth / Payment / Chat / Hosting categories. Detection evidence is now first-class — every `Detection` carries `Vec<Evidence>` explaining why it fired (header / cookie / body marker / TLS hash / DNS / ASN / peer-cert / etc.), and `Confidence::{High, Medium, Low}` is **derived** deterministically from summed weights (rule lives in `fingerprint::detection::Confidence::from_evidence`).

The three legacy types are marked `#[deprecated]` (slices B5, B7) but still compile for one release; their `From` conversions to `fingerprint::Vendor` are total. Production callers (`policy::engine`, `runner::JobRunner`) continue to operate during the deprecation window. Final deletion of `discovery::tech_fingerprint` / `runner::ChallengeDetector` / `antibot::detect_*` / `antibot::block_detector` is scheduled for the next release after every in-tree caller has migrated. The non-detection halves of `antibot/` (`bypass`, `cookie_pin`, `solver`, `telemetry`, `recaptcha`) stay — those are action modules, not detection, and were never in scope for this consolidation. ADR-0004 records the parallel decision to delete `runner::AutoFetcher`.
