## Per-site notes

- **antoinevastel.com** — pass: bot detection page rendered, all our
  signals coherent (headless not declared, navigator clean).
- **arh.antoinevastel.com** — pass: AreYouHeadless reports
  `not-headless`. Stealth shim coherent.
- **pixelscan.net** — partial: Page loaded successfully (1035 KB
  screenshot saved), but pixelscan's client-side detector flagged some
  inconsistency.
- **5 unreachable sites** — all 5 hit `navigate: Request timed out.`
  Curl-from-same-host shows TTFB ~0.4-1.4s for each. Not a network
  reachability issue. Two hypotheses tracked for v1.1:
  - These pages keep the `load` lifecycle event pending due to
    long-running JS / third-party widgets. The new
    `--navigation-lifecycle domcontentloaded` flag (or
    `CRAWLEX_NAVIGATION_LIFECYCLE=domcontentloaded` env) should
    switch the watcher to `DOMContentLoaded`. Validator may not be
    picking up the env var.
  - Cargo-test env propagation gap: the env-driven switch in
    `NavigationWatcher::until_page_load` reads at first navigation;
    if cargo test forks a sub-shell that strips env, the constant
    30s default still applies. v1.1 will move this to a `Config`
    field plumbed through the validator harness so it's not
    env-dependent.
